# 📖 Changelog

All notable changes to the **Jarvis AI Assistant** project will be documented in this file.

This project follows the principles of **Keep a Changelog** and **Semantic Versioning (SemVer)**.

---

# Versioning

Version numbers follow the format:

```text
MAJOR.MINOR.PATCH
```

* **MAJOR** – Incompatible API or architecture changes.
* **MINOR** – New features added in a backward-compatible manner.
* **PATCH** – Bug fixes, documentation updates, and performance improvements.

---

# Release Types

| Type       | Description                    |
| ---------- | ------------------------------ |
| Added      | New features                   |
| Changed    | Updates to existing features   |
| Deprecated | Features scheduled for removal |
| Removed    | Deleted functionality          |
| Fixed      | Bug fixes                      |
| Security   | Security improvements          |

# Upgrade Notes

When upgrading between major versions:

* Read the release notes.
* Review configuration changes.
* Update project dependencies.
* Backup custom settings.
* Test new functionality before production use.

Major releases may include architectural improvements that require manual configuration updates.

---

# Deprecation Policy

Deprecated features will:

1. Be documented in advance.
2. Remain functional for at least one minor release where practical.
3. Include migration guidance.
4. Be removed in a future major version.

This policy helps users transition smoothly.

---

# How to Read This Changelog

Each release is organized into:

* **Added** – New functionality
* **Changed** – Modifications to existing behavior
* **Deprecated** – Features that will be removed
* **Removed** – Deleted functionality
* **Fixed** – Bug fixes
* **Security** – Security-related updates

This format makes it easy to understand the impact of each release.

# Contributing to the Changelog

When submitting a Pull Request that changes functionality, contributors should update this changelog by adding an entry under the **[Unreleased]** section.

Maintainers will organize these entries into the appropriate release during the release process.
