# 📜 Code of Conduct

## Our Pledge

We, as contributors, maintainers, and community members, pledge to make participation in the **Jarvis AI Assistant** project a welcoming, respectful, and harassment-free experience for everyone.

We are committed to fostering an open, inclusive, and collaborative environment regardless of:

* Age
* Body size
* Disability
* Ethnicity
* Gender identity or expression
* Level of experience
* Nationality
* Personal appearance
* Race
* Religion
* Sexual identity and orientation
* Technical background

Everyone should feel safe and respected while contributing to this project.

---

# Our Standards

Examples of behavior that contributes to a positive community include:

* Being respectful and kind.
* Welcoming new contributors.
* Providing constructive feedback.
* Accepting feedback gracefully.
* Helping others learn.
* Respecting different opinions and experiences.
* Writing clear and professional communication.
* Focusing discussions on improving the project.

---

# Unacceptable Behavior

The following behaviors are not acceptable within the project community:

* Harassment or intimidation.
* Personal attacks.
* Discrimination of any kind.
* Hate speech.
* Offensive jokes or comments.
* Trolling or deliberately disruptive behavior.
* Publishing private information without permission.
* Threats or violent language.
* Spam or repeated unwanted messages.
* Deliberately misleading contributors.

These behaviors may result in moderation or removal from the community.

---

# Communication Guidelines

When participating in discussions:

* Assume good intentions.
* Discuss ideas rather than individuals.
* Be patient with beginners.
* Ask questions respectfully.
* Keep conversations focused on the project.
* Provide evidence when reporting bugs or proposing changes.

Healthy discussions help improve both the software and the community.

---

# Code Review Expectations

When reviewing code:

* Review the code, not the contributor.
* Explain suggested improvements clearly.
* Be respectful when requesting changes.
* Recognize good work.
* Encourage learning and collaboration.

Constructive reviews create a stronger project.

---

# Responsibilities of Maintainers

Project maintainers are responsible for:

* Enforcing this Code of Conduct.
* Treating all contributors fairly.
* Reviewing reports objectively.
* Taking appropriate action when necessary.
* Maintaining a welcoming environment.
* Encouraging positive collaboration.

Maintainers should act consistently and transparently.

---

# Scope

This Code of Conduct applies to all project spaces, including:

* GitHub Issues
* Pull Requests
* Discussions
* Documentation contributions
* Community chats related to the project
* Any official communication channels

It also applies when someone is representing the project in public.

---

# Reporting Issues

If you experience or witness behavior that violates this Code of Conduct:

1. Gather relevant information.
2. Document the incident if possible.
3. Contact the project maintainers through GitHub.
4. Provide enough detail for the maintainers to investigate.

Reports will be reviewed respectfully and, where practical, confidentially.

---

# Enforcement

Project maintainers may take actions including:

* Private warning
* Request for behavior change
* Temporary suspension
* Removal of inappropriate content
* Permanent ban from contributing

The response will depend on the severity and frequency of the behavior.

---

# Our Community Values

The Jarvis AI Assistant community values:

* Respect
* Honesty
* Curiosity
* Collaboration
* Inclusivity
* Transparency
* Continuous learning
* Privacy
* Security
* Open-source development

These values guide the direction of the project and its community.

---

# Diversity and Inclusion

We believe that diverse perspectives lead to better software.

Contributors from all backgrounds and experience levels are encouraged to participate, ask questions, and share ideas.

No one should feel excluded because they are new to programming, open source, or artificial intelligence.

---

# Conflict Resolution

When disagreements arise:

* Remain respectful.
* Focus on facts and technical reasoning.
* Seek compromise where appropriate.
* Involve maintainers if necessary.
* Avoid escalating conflicts publicly.

Professional communication helps resolve issues more effectively.

---

# Open Source Spirit

Open source thrives through collaboration.

Whether your contribution is:

* Reporting a bug
* Improving documentation
* Writing code
* Designing graphics
* Reviewing pull requests
* Suggesting new ideas

your effort is appreciated and helps move the project forward.
